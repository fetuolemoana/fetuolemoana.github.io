Design to web

Responsive websites

Responsive websites are designed to work on any sized screen and / or window; they are flexible via the use of percentages and not fixed widths. When you resize a window, the website content will adjust (or remove things like large images) to fit without cutting stuff out awkwardly or making things too small to read or interact with.

This is important because more people will be able to access a good quality experience of a website and only one website will need to be created, and not multiple ones for different sized devices.

Mobile first design

Clarissa Peterson’s presentation on designing responsive websites likens designing for desktop first and then mobile to moving from a large house to a small apartment. There will be ‘boxes’ that just won’t fit in the apartment / mobile screen.

Anything that can be viewed on a smaller screen can be viewed on a larger screen; this is not the case the other way around. Also, mobile use is increasing while desktop use is declining.

Other positive impacts of designing for mobile first includes content becoming more succinct and information better organised.

Frameworks pros and cons

Frameworks provide a common website structure so that developers don’t have to repeatedly rewrite code from scratch. This saves a lot of time. However, one of the disadvantages of this is that learners may not learn how to create frameworks at all, which could lead to less depth of understanding of how they work.

Using wireframes

Creating wireframes is part of good design and development process; they are a visual blueprint for a website and focus on site layout, how elements will be organised and navigation. It is used as a visual representation of the design plan and helps us to direct and see where we want to go with the website design.
