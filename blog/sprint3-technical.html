HTML and CSS meet JavaScript

An analogy to describe the difference between HTML and CSS

HTML holds the structure and textual content of a web page and CSS provides the instructions on its style and appearance. For example, the HTML could include headings, subheadings and paragraph text; and the CSS allows us to apply a different colour or font to that text.

The practice of tatau in Samoa may be used as an analogy for the differences between HTML and CSS. The malu and pe'a feature particular structures and motifs, and the ink is used as the content filling in this structure. Similarly, HTML holds the content and structure of a web page.

The ways the structure and ink adorn / style the body of the malu or pe'a holder, is decided largely by the tufuga (and the wearer's holding ability), and the ancestors. Perhaps instead of cascading style sheets, we would describe it in a tatau context, a collective spirit song.

Explain control flow and loops using an example process from everyday life

Control flow is about the order a computer executes what you ask it to do using coding languages. Generally, computer instructions are carried out from top to bottom. However, looping statements such as for, while and do-while, can require a computer to disrupt a default top down flow.

The process of wiping blood while tatau is being carried out is a life process that can be compared to control flow and looping.

  1. Skin is stretched
  2. the au is tapped
  3. the sharp bone hits the flesh
  4. if the amount of blood is less than x, return to step 1
  5. else, assistant wipes the blood.

Describe what the DOM is and an example of how you might interact with it

The Document Object Model (DOM) is the browser's understanding and representation of an HTML document. The DOM enables JavaScript to be used to access the HTML as an object and change the HTML elements of a web page; allowing us to make more dynamic and interactive websites. You can interact with the DOM by opening developer tools in the browser and navigating to the 'elements' section, this is the DOM.

Explain the difference between accessing data from arrays and objects

Objects are variables that can contain multiple items (values, properties, methods / functions). You can access these things inside the objects by using the name of the property or method like so:

  • objectName.propertyName
  • objectName["propertyName"]
  • objectName.methodName()

Arrays can store multiple items (elements) under a single label (variable) too. However, the items in an array are accessed or 'called' using an automatically given index number starting from 0. To access an array item you may use the variable followed by the index number in brackets e.g. arrayName[0] will access the first item of the array.

Explain what functions are and why they are useful

A function is a set of instructions / a customised JavaScript command created to carry out a particular action. Helpfully, the Watch and Code Practical JavaScript course describes functions as 'recipes'. The basic structure of a function is as follows:

function nameOfFunction(parameters) {
instructions;
}
To run a function:
nameOfFunction()

A function can be defined once and reused many times; this helps us to avoid being DRY (don't repeat yourself) and repeating ourselves again and again and again.
